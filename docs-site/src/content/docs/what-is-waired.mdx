AI coding tools normally send everything you type to a company's servers, and
charge you per message. Waired lets **one computer you already own** run the AI
instead, and lets **your other devices use it from anywhere**.

## What you get

- **No per-message bill.** The AI runs on hardware you already paid for.
- **Your prompts stay yours.** They travel between your own devices over an
  encrypted link. Waired's servers introduce your devices to each other and
  never see what you send.
- **Your tools don't change.** Claude Code, OpenCode, and any
  OpenAI-compatible app keep working exactly as before.
- **It works away from home.** Your laptop reaches your desktop's AI from a
  café or an office, without you setting up a VPN or opening ports.

## What it is not

- **Not a chat app.** Waired has no chat window of its own. It makes the tools
  you already use talk to your own AI instead of a company's.
- **Not a way to run huge frontier models.** What you can run is bounded by
  your hardware. A good desktop runs a capable coding model; a thin laptop runs
  a small one. Waired tells you honestly which one fits.
- **Not something that hides fallbacks.** If your own AI cannot answer, the
  request goes to the cloud provider **and Waired says so** — it never pretends
  a cloud answer came from your machine.

## What it costs to run

Nothing per message. You need one computer that stays on when you want to use
it, and enough memory for the model — see the
[model catalog](/reference/model-catalog/) for the numbers.

<Aside type="note" title="Do I need a graphics card?">
	No. A graphics card makes answers much faster, and if you have one you should
	use that computer. But a recent processor runs a small model well enough to
	be useful.
</Aside>

## How it works, briefly

Every computer you sign in on joins a private network of your own. When you ask
a question, Waired decides where it should run — this computer, another of your
computers, or (visibly) the cloud — and sends it there over an encrypted
connection. Your devices talk **directly** to each other whenever the network
allows; when a firewall gets in the way, an encrypted [relay](/reference/glossary/#relay) passes the sealed
data along without being able to read it.
