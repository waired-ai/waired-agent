Run this once on every computer you want in your network.

<Aside type="tip" title="Same account = same network">
	Sign in to **every** computer with the **same Google account**. That shared
	sign-in is what lets one machine use another's AI. Different accounts make
	separate, unconnected networks.
</Aside>

## Start setup

<Tabs syncKey="os">
	<TabItem label="Windows">
		Open **PowerShell as Administrator** (right-click the Start button →
		**Terminal (Administrator)**), then run:

		```powershell
		& "C:\Program Files\Waired\waired.exe" init
		```
	</TabItem>
	<TabItem label="macOS">
		```sh
		sudo waired init
		```
	</TabItem>
	<TabItem label="Linux">
		```sh
		sudo waired init
		```
	</TabItem>
</Tabs>

It prints a sign-in link and waits for you to press **Enter**, which opens it
in your browser. You can also copy the link into any browser yourself.

<Aside type="caution" title="Leave that window open until setup finishes">
	Setup continues in the browser, but this window is still doing the parts that
	need administrator rights — installing the AI software and connecting your
	coding tools.

	If you close it, the browser tells you so (*“The setup command on … was closed
	before this finished. Your progress was saved.”*) — run the same command again
	to carry on. Nothing is lost, and the model download continues regardless.
</Aside>

## Finish in the browser

After you sign in, your browser lands on the setup page for the computer you
are setting up. It shows what that computer is doing, live:

<Screenshot
	src="setup-wizard-progress.png"
	alt="The Waired setup page in a browser, showing three steps — Install the AI software (done), Download the AI model (in progress, with a bar reading 7.9 GB of 19.2 GB), and Check the speed (waiting)."
	caption="Three steps, reported by the computer itself. Closing this tab does not stop them."
/>

| Step | What is happening | How long |
|---|---|---|
| **Install the AI software** | The program that loads and runs models. Needs administrator rights — which is why the terminal window has to stay open. | A minute or two |
| **Download the AI model** | The AI itself. Waired picks the best one this computer can actually run. | 5–30 minutes |
| **Check the speed** | A short measurement, so Waired knows whether this machine keeps up. | Up to 5 minutes |

Then you are asked which coding tools to connect. Turn on **Claude Code** if you
use it — you can change this later.

When it finishes, the browser lands on the device's page, where everything can
be seen and changed afterwards.

<Aside type="note" title="The setup page is in English">
	The web console is English-only for now. Translations are planned.
</Aside>

## The choices you are making

Two settings decide what this computer is *for*. Setup picks sensible defaults
from its hardware, and either can be changed at any time afterwards.

**Does this computer run AI models?**
: Yes on your desktop or GPU box. No on a laptop that should only *use* another
  computer's AI — it still joins your network, it just does not host a model.
  Waired defaults to No on machines too small to run anything useful, and tells
  you what they would need.

**Can your other devices use this computer's AI?**
: Yes on a machine meant to serve your laptop and phone. No keeps the AI to this
  machine — you can still use it here. This is independent of the first setting;
  see [sharing vs. pausing](/reference/cli/#sharing-vs-pausing).

<Aside type="caution" title="“This machine can only run a very small model”">
	On a low-memory computer, setup may report that only a very small model fits
	and suggest **not** running one. Take that advice: at that size a coding
	model produces broken output more often than useful output. The computer is
	still worth setting up — it can use the AI on your other machines.
</Aside>

## Set up from the terminal instead

The browser flow is a convenience, not a requirement. The terminal flow runs
automatically when there is no browser to open — a server over SSH, a machine
with no desktop, `--no-browser`, or `--non-interactive` — and you can choose it
at any time by pressing **Enter** at:

```
Setup is continuing in your browser…
(press Enter to continue in the terminal instead)
```

It asks the same things as questions:

<Steps>

1. **Google sign-in.** The link is printed; press Enter to open it. On a
	headless machine only the URL and a pairing code are printed — open them on
	another device.

2. **Run AI models on this computer? (Y/n)** — the first choice above. The
	default follows this machine's memory.

3. **Let your other devices use this computer's AI? (Y/n)** — the second choice
	above.

</Steps>

If you enabled models, it then picks one that fits, checks there is disk space
for it (stepping down to a smaller model rather than filling the disk), and
downloads it in the foreground with a progress bar:

```
Starting the inference engine…
Preparing to download qwen3-coder-30b-a3b-instruct…
Downloading qwen3-coder-30b-a3b-instruct: 41%  7.9 GB / 19.2 GB (44 MB/s)
```

Then it measures the speed and, if this computer cannot keep up, offers a
lighter model — naming both, with their quality tiers, so you can weigh the
trade-off. Finally it asks whether to route Claude Code through Waired now.
Answering no leaves the integration installed but inactive; turn it on later
with `sudo waired claude enable`.

**You should see**:

```
╭─ 🎉 Waired is ready — everything completed successfully! ─────────╮
│  Account   you@example.com                                       │
│  Device    my-desktop  d-4f2a…                                   │
│  Network   personal  overlay 100.64.0.3                          │
│  Model     qwen3-coder-30b-a3b-instruct  (58 tok/s)              │
│  Gateway   http://127.0.0.1:9479                                 │
│  Claude    http://127.0.0.1:9473 (managed settings)              │
│                                                                  │
│  Point your coding agent at Waired and start building.           │
╰──────────────────────────────────────────────────────────────────╯
```

<details>
<summary>Options for servers and scripts</summary>

| Flag | Effect |
|---|---|
| `--no-browser` | Never open a browser; print the URL and pairing code |
| `--non-interactive` | Accept the hardware-derived defaults, ask nothing |
| `--no-wait-model` | Return to the shell immediately; download in the background |
| `--inference-bundled-model-id <id>` | Pin a model instead of letting Waired choose |
| `--inference-enabled=true` | Run models even on a machine Waired rated too small |
| `--control <url>` | Point at a non-default service |
| `--mask-pii` | Mask home directory, username, hostname and email in the output, for pasting into a bug report |

</details>

## Setting up again

Running setup again on a computer that is already signed in is safe: it
re-authenticates and reopens the setup page, and does **not** count as a new
device against your account.
