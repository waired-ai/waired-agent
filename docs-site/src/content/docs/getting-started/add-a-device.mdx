Once one computer runs the AI, every other computer you sign in on can use it —
at home, at the office, or on café Wi-Fi.

<Aside type="tip" title="The whole trick is the account">
	Sign in with the **same Google account** you used on the first computer.
	That is what puts them on the same private network. There is nothing to pair
	and no address to copy.
</Aside>

## On the new computer

<Steps>

1. **Install Waired** — [Windows](/getting-started/install/windows/) ·
	[macOS](/getting-started/install/macos/) ·
	[Linux](/getting-started/install/linux/)

2. **Sign in** the same way as the first time, with the same Google account:
	see [Sign in and set up](/getting-started/first-run/).

3. **Say no to running models**, unless this computer is also powerful enough to
	host one. A laptop that only *uses* the AI does not need to download
	anything — it will be answered by the computer you set up first.

</Steps>

## Check that it can reach the first computer

```sh
waired status --observability
```

Look at the **Mesh** line:

```
Mesh:     2 enrolled / 1 reachable / 1 ready
```

- **enrolled** — how many computers are in your network
- **reachable** — how many this computer can currently talk to
- **ready** — how many of those have a model loaded and are offering it

`1 reachable / 1 ready` on a laptop means your AI computer is up and available.
If it reads `0 reachable`, see
[A device can't reach my AI computer](/troubleshooting/#my-other-computer-cannot-reach-the-ai).

Then ask it something:

```sh
waired infer "say hi"
```

The answer comes from the other computer, over an encrypted connection.

## Where the answer comes from

By default Waired chooses for you: it uses this computer's own AI if it has one,
and otherwise asks one of your other computers. To pin it down:

```sh
waired worker get                    # where requests go now
waired worker set --mode=local-only  # never use another computer
waired worker set --mode=peer-preferred
waired peers list                    # see your other computers
waired worker set --pin=<device>     # always use this one
```

<Aside type="note" title="Nothing to open on your router">
	Your computers connect directly when the network allows it, and fall back to
	an encrypted relay when a firewall gets in the way. Both are automatic — no
	port forwarding, no VPN configuration.
</Aside>

## Remove a device later

Uninstalling Waired on a computer removes it from your list automatically. To
remove one you no longer have access to, open your device list at
[app.waired.ai](https://app.waired.ai) and delete it there.
