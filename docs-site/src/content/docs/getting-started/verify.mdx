Three checks, quickest first. If any of them fails, `waired doctor` is the one
command to run next.

## 1. Look at the Waired icon

Click the **Waired icon** — in the menu bar on macOS, next to the clock on
Windows and Linux.

<Screenshot
	src="app-ready.png"
	alt="The Waired menu showing the signed-in account, a Connected status, and an Inference submenu naming the active model."
	caption="Signed in, connected, and a model loaded."
	width="26rem"
/>

**What good looks like:** your account address at the top, a connected state,
and — on a computer that runs models — **Inference** naming the active model.

## 2. Ask it something

```sh
waired infer "say hi"
```

**What good looks like:** a greeting, within a few seconds.

This goes straight to the model on this computer, so it works even if nothing
else on your network is reachable. It is the fastest way to prove the AI itself
is alive.

To see *where* the answer came from and why:

```sh
waired infer --explain "say hi"
```

## 3. Read the full status

```sh
waired status --observability
```

**What good looks like**, on a computer that runs models:

```
Observability:
  Device:   my-desktop  uptime: 1m23s
  Engine:   ready (model=qwen3-coder-30b-a3b-instruct, 0/4 slots used, inflight=0)
  Share:    enabled    Paused: no
  Mesh:     2 enrolled / 1 reachable / 1 ready
  Last:     -  (no inference requests yet)
```

| Line | What to check |
|---|---|
| **Engine** | `ready` means a model is loaded and can answer. `not ready` usually means it is still downloading — see [No answer comes back](/troubleshooting/#no-answer-comes-back). |
| **Share** | `enabled` means your other devices may use this computer's AI. Off is fine if you only use it here. |
| **Paused** | `no` means Waired is handling requests. `yes` — run `waired resume`. |
| **Mesh** | `enrolled / reachable / ready` across your computers. On a laptop, this is where you confirm your AI machine is up. |

Add `-o json` for a machine-readable version.

<Aside type="tip" title="On a laptop that does not run models">
	`Engine: not ready` is correct there — that computer is not supposed to host
	a model. What matters is the **Mesh** line showing at least `1 ready`, which
	means one of your other computers is available to answer.
</Aside>
