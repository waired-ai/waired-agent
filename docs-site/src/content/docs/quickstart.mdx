By the end of this page you will have an AI model running on your own
computer, and Claude Code answering from it instead of from the cloud.

## Before you start

|  | |
|---|---|
| **Time** | About 10 minutes of your attention, plus 5–30 minutes of unattended downloading. |
| **You need** | One computer to run the AI on (Windows 11, macOS 13+, or Ubuntu 24.04+/Debian trixie+), a Google account, and administrator rights on that computer. |
| **You don't need** | A graphics card. A GPU makes answers faster, but a recent CPU runs a small model fine. |

<Aside type="tip" title="Which computer should run the AI?">
	The most powerful one you own — a desktop with a graphics card, or a Mac with
	Apple Silicon. Your laptop can use that computer's AI later, from anywhere,
	without running anything itself. If you only have one computer, use it: it
	will both run the AI and use it.
</Aside>

<Steps>

1. ### Install Waired

	<Tabs syncKey="os">
		<TabItem label="Windows">
			**Download the installer** and double-click it:

			<LinkCard
				title="WairedSetup-x64.exe"
				href="https://github.com/waired-ai/waired-agent/releases/latest"
				description="From the latest release page — the file named WairedSetup-<version>-x64.exe."
			/>

			Windows will ask for permission to make changes (a blue
			**User Account Control** window) — choose **Yes**.

			<Aside type="caution" title="“Windows protected your PC”">
				Our installer is not code-signed yet, so Windows SmartScreen shows a
				blue warning the first time. Click **More info**, then
				**Run anyway**. If you would rather not, use the command below —
				it downloads the same files.
			</Aside>

			<details>
			<summary>Prefer a command? (PowerShell)</summary>

			```powershell
			iwr -useb https://github.com/waired-ai/waired-agent/releases/latest/download/install.ps1 | iex
			```

			</details>
		</TabItem>
		<TabItem label="macOS">
			macOS has no double-click installer yet. Open **Terminal**
			(press ⌘ + Space, type `Terminal`, press Enter), paste this line,
			and press Enter:

			```sh
			curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh
			```

			You will be asked for your Mac password. Nothing is echoed as you
			type it — that is normal; type it and press Enter.
		</TabItem>
		<TabItem label="Linux">
			Open a terminal, paste this line, and press Enter:

			```sh
			curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh
			```

			You will be asked for your password. Nothing is echoed as you type
			it — that is normal; type it and press Enter.

			<Aside type="note">
				The one-line installer supports Ubuntu 24.04+ and Debian trixie+.
				Other distributions are not covered yet — see
				[Install on Linux](/getting-started/install/linux/).
			</Aside>
		</TabItem>
	</Tabs>

	Before it changes anything, the installer prints a short summary of what it
	is about to do and waits for you to answer `Proceed? [Y/n]` — press Enter to
	continue.

2. ### Start setup

	This is the only command you have to type. Everything after it happens in
	your browser.

	<Tabs syncKey="os">
		<TabItem label="Windows">
			Open **PowerShell as Administrator** (right-click the Start button →
			**Terminal (Administrator)**), then run:

			```powershell
			& "C:\Program Files\Waired\waired.exe" init
			```
		</TabItem>
		<TabItem label="macOS">
			```sh
			sudo waired init
			```
		</TabItem>
		<TabItem label="Linux">
			```sh
			sudo waired init
			```
		</TabItem>
	</Tabs>

	It prints a sign-in link and waits. Press **Enter** to open it, and sign in
	with Google.

	<Aside type="caution" title="Leave this window open">
		Setup moves to your browser, but this window is still doing the work that
		needs administrator rights. **Do not close it** until setup finishes. If
		you close it by accident, nothing breaks and no progress is lost — you
		just run the same command again to pick up where you left off.
	</Aside>

3. ### Finish setup in your browser

	After you sign in, the browser takes you straight to the setup page for the
	computer you just signed in on. It works through three steps on its own:

	<Screenshot
		src="setup-wizard-progress.png"
		alt="The Waired setup page in a browser, showing three steps — Install the AI software (done), Download the AI model (in progress, with a bar reading 7.9 GB of 19.2 GB), and Check the speed (waiting)."
		caption="The setup page reports what your computer is actually doing. You can close the tab during the download — it keeps going."
	/>

	- **Install the AI software** — the program that runs models, installed for
	  you.
	- **Download the AI model** — a few gigabytes, so this is the slow step.
	  Waired picks the best model this computer can actually run; you do not
	  have to choose one.
	- **Check the speed** — a short measurement, up to about five minutes.

	Then it offers to connect your coding tools. Turn on **Claude Code**.

	<Aside type="tip" title="You can close the browser tab">
		The download continues without it. Come back to
		[app.waired.ai](https://app.waired.ai) and open the device to see where it
		got to. (Closing the *terminal* window from step 2 is the one thing that
		stops setup — see above.)
	</Aside>

	Meanwhile the terminal window shows:

	```
	Setup is continuing in your browser…
	(press Enter to continue in the terminal instead)
	```

	Pressing Enter there switches the whole thing back to question-and-answer in
	the terminal — useful if the browser did not open. See
	[Set up from the terminal instead](/getting-started/first-run/#set-up-from-the-terminal-instead).

4. ### Ask it something

	Back in the terminal:

	```sh
	waired infer "say hi"
	```

	A greeting should come back within a few seconds. That answer was produced
	on your own computer — nothing was sent to a cloud AI provider.

	<Aside type="note" title="Nothing came back?">
		Run `waired doctor`. It checks every part of the setup, marks each one
		✓ / ⚠ / ✗, and — if you press **f** — repairs what it can. That single
		command resolves most first-run problems.
	</Aside>

5. ### Use it from Claude Code

	Open Claude Code as you normally would, in any project, and ask it
	something. The footer shows which AI answered.

	<Screenshot
		src="claude-code-statusline.png"
		alt="The Claude Code footer showing the Waired status line, naming the local model that answered the last request."
		caption="Claude Code's footer tells you which AI answered — yours, or the cloud."
	/>

	To check, or to switch back and forth:

	```sh
	waired claude route          # show where Claude Code is running
	waired claude route waired   # your own AI only
	waired claude route anthropic # the real Anthropic API
	```

	Inside a Claude Code session, `/waired-route` does the same thing without
	leaving the conversation.

</Steps>
