By the end of this page you will have an AI model running on your own
computer, and Claude Code answering from it instead of from the cloud.

## Before you start

|  | |
|---|---|
| **Time** | About 10 minutes of your attention, plus 5–30 minutes of unattended downloading. |
| **You need** | One computer to run the AI on (Windows 11, macOS 13+, or Ubuntu 24.04+/Debian trixie+), a Google account, and administrator rights on that computer. |
| **You don't need** | A graphics card. A GPU makes answers faster, but a recent CPU runs a small model fine. |

<Aside type="tip" title="Which computer should run the AI?">
	The most powerful one you own — a desktop with a graphics card, or a Mac with
	Apple Silicon. Your laptop can use that computer's AI later, from anywhere,
	without running anything itself. If you only have one computer, use it: it
	will both run the AI and use it.
</Aside>

## Step 1 — Install Waired

Installing is one command. Open a terminal, paste the line for your system, and
press Enter.

<Tabs syncKey="os">
	<TabItem label="Windows">
		Open **PowerShell as Administrator**: right-click the Start button →
		**Terminal (Administrator)** → **Yes**.

		```powershell
		iwr -useb https://github.com/waired-ai/waired-agent/releases/latest/download/install.ps1 | iex
		```

		`iwr` downloads Waired's install script, `iex` runs it.
	</TabItem>
	<TabItem label="macOS">
		Open **Terminal**: press ⌘ + Space, type `Terminal`, press Enter.

		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh
		```

		`curl` downloads Waired's install script, `sh` runs it. You will be asked
		for your Mac password — **nothing appears on screen as you type it**,
		which is normal.
	</TabItem>
	<TabItem label="Linux">
		Open a terminal.

		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh
		```

		`curl` downloads Waired's install script, `sh` runs it. You will be asked
		for your password — **nothing appears on screen as you type it**, which
		is normal.
	</TabItem>
</Tabs>

Before changing anything, the installer prints a summary of what it is about to
do and asks `Proceed? [Y/n]` — press Enter to continue.

<Aside type="tip" title="Would rather read the script first?">
	Download it, read it, run it — see the platform pages:
	[Windows](/getting-started/install/windows/#read-the-script-before-running-it) ·
	[macOS](/getting-started/install/macos/) ·
	[Linux](/getting-started/install/linux/).
</Aside>

That is the only command you *have* to type. From here, choose how you want to
finish.

## Step 2 — Finish setup

<Tabs syncKey="mode">
	<TabItem label="On screen">

		{/* VERIFY BEFORE PUBLISHING: tray-initiated sign-in exists

		    waired-agent#116. Confirm on all three OSes. */}

		<Steps>

		1. **Click the Waired icon** in your menu bar (macOS) or system tray
			(Windows, Linux), and choose **Log in…**.

			<Screenshot
				src="tray-not-signed-in.png"
				alt="The Waired menu open, showing “○ Not signed in” at the top and a “Log in…” entry below it."
				caption="Before you sign in, the menu says “Not signed in”."
				width="26rem"
			/>

		2. **Sign in with Google** in the browser window that opens.

		3. **Watch the browser finish setup.** It installs the AI software,
			downloads a model, and measures this computer's speed — see
			[what those three steps are](#what-the-three-setup-steps-are).

			<Screenshot
				src="setup-wizard-progress.png"
				alt="The Waired setup page in a browser, showing three steps — Install the AI software (done), Download the AI model (in progress, 7.9 GB of 19.2 GB), and Check the speed (waiting)."
				caption="You can close this tab — the download keeps going."
			/>

		4. **Turn on Claude Code** when it offers to connect your coding tools.

		</Steps>

		<Aside type="caution" title="If setup asks you to run something">
			Some steps need administrator rights, and the browser cannot grant
			them. If the page asks you to go back to the computer and run a
			command, switch to the **In the terminal** tab above and start from
			`waired init` — nothing you have done is lost.
		</Aside>

	</TabItem>
	<TabItem label="In the terminal">

		<Steps>

		1. **Start setup:**

			<Tabs syncKey="os">
				<TabItem label="Windows">
					```powershell
					& "C:\Program Files\Waired\waired.exe" init
					```
					(from the same administrator PowerShell window)
				</TabItem>
				<TabItem label="macOS">
					```sh
					sudo waired init
					```
				</TabItem>
				<TabItem label="Linux">
					```sh
					sudo waired init
					```
				</TabItem>
			</Tabs>

		2. **Sign in.** It prints a link and waits — press **Enter** to open it,
			then choose your Google account.

		3. **Answer two questions:**

			- *Run AI models on this computer?* — **Y** on the computer you
			  picked above; **N** on a laptop that should only use another
			  computer's AI.
			- *Let your other devices use this computer's AI?* — **Y** so your
			  laptop can reach it.

		4. **Wait.** Waired installs the AI software, picks a model that fits
			this machine, downloads it with a progress bar, and measures the
			speed — see [what those three steps are](#what-the-three-setup-steps-are).

			```
			Starting the inference engine…
			Preparing to download qwen3-coder-30b-a3b-instruct…
			Downloading qwen3-coder-30b-a3b-instruct: 41%  7.9 GB / 19.2 GB (44 MB/s)
			```

		5. **Say Y** to routing Claude Code through Waired.

		</Steps>

		**You should see**:

		```
		╭─ 🎉 Waired is ready — everything completed successfully! ─────────╮
		│  Account   you@example.com                                       │
		│  Device    my-desktop  d-4f2a…                                   │
		│  Network   personal  overlay 100.64.0.3                          │
		│  Model     qwen3-coder-30b-a3b-instruct  (58 tok/s)              │
		│  Gateway   http://127.0.0.1:9479                                 │
		│  Claude    http://127.0.0.1:9473 (managed settings)              │
		│                                                                  │
		│  Point your coding agent at Waired and start building.           │
		╰──────────────────────────────────────────────────────────────────╯
		```

	</TabItem>
</Tabs>

<Aside type="note" title="You can mix the two">
	They are the same setup seen from two places. Starting in the terminal opens
	the browser page anyway (press **Enter** at *“Setup is continuing in your
	browser…”* to stay in the terminal instead), and anything set in one shows up
	in the other. On a server with no desktop, the terminal is the only option.
</Aside>

### What the three setup steps are

| Step | What is happening | How long |
|---|---|---|
| **Install the AI software** | The program that loads and runs models. Needs administrator rights. | A minute or two |
| **Download the AI model** | The AI itself. Waired picks the best one this computer can actually run — you do not have to choose. | 5–30 minutes |
| **Check the speed** | A short measurement, so Waired knows whether this machine keeps up. If it cannot, you are offered a lighter model. | Up to 5 minutes |

## Step 3 — Ask it something

<Tabs syncKey="mode">
	<TabItem label="On screen">
		Click the **Waired icon** again. It should show your account, and
		**Inference** should name the model that is running.

		<Screenshot
			src="tray-ready.png"
			alt="The Waired menu showing the signed-in account, a Connected status, and an Inference submenu naming the active model."
			caption="Signed in, connected, and a model loaded — that is a working setup."
			width="26rem"
		/>
	</TabItem>
	<TabItem label="In the terminal">
		```sh
		waired infer "say hi"
		```

		A greeting should come back within a few seconds. That answer was
		produced on your own computer — nothing was sent to a cloud AI provider.
	</TabItem>
</Tabs>

<Aside type="note" title="Nothing came back?">
	Run `waired doctor`. It checks every part of the setup, marks each one
	✓ / ⚠ / ✗, and — if you press **f** — repairs what it can. That single
	command resolves most first-run problems.
</Aside>

## Step 4 — Use it from Claude Code

Open Claude Code as you normally would, in any project, and ask it something.
The footer tells you which AI answered.

<Screenshot
	src="claude-code-statusline.png"
	alt="The Claude Code footer showing the Waired status line, naming the local model that answered the last request."
/>

Type `/waired-route` in the conversation to switch between your own AI and the
cloud, or from a terminal:

```sh
waired claude route           # show where Claude Code is running
waired claude route waired    # your own AI only
waired claude route anthropic # the real Anthropic API
```
