Waired は、すでに使っているコーディングエージェントを自分のモデルに向けます。要素は
2 つあります:

- **`waired link`** はユーザーごとの連携ファイル — Claude Code のスキル
  (`~/.claude/skills/`) と OpenCode のプロバイダープラグイン
  (`~/.config/opencode/plugin/waired.js`) — をセットアップし、それらが
  使用するゲートウェイトークンを発行します。
- **マネージド設定 (managed settings)** は、Claude Code のシステム全体の
  `managed-settings.json` を書き込むことで、Claude Code を自分のモデルへ
  向けます。そのため、環境変数やシェルエイリアスを一切設定する必要はありません。

## Claude Code

Claude Code は**マネージド設定 (managed settings)** によってルーティングされます。
Waired は Claude Code のシステム全体の `managed-settings.json` を書き込み、
Claude Code が自分のローカルモデルと通信するようにします。資格情報はそこに
書き込まれないため、Claude Code は既存の claude.ai サブスクリプションを
そのまま使い続けます — サブスクリプションの auto モード (opusplan / Max の
Opus→Sonnet 使用量フォールバック) も含めて維持されます。**証明書も `/etc/hosts`
の編集もシェルの変更もありません**。これは Linux・macOS・Windows で同じように
動作します。

対話的な `waired init` では、エンジンのセットアップと初回モデルのダウンロードが
終わった**インストールの最後**に、これを有効化するかどうかを確認します
(非対話インストールでは冒頭の統合 consent で有効化されます)。直接管理する
こともできます。

<Steps>
1. 連携ファイルとゲートウェイトークンをセットアップします:

   ```sh
   waired link claude-code
   ```

2. マネージド設定が有効になっていることを確認します (通常は `waired init` が済ませています):

   <Tabs syncKey="os">
   	<TabItem label="Linux/macOS">
   		```sh
   		sudo waired claude enable
   		```
   	</TabItem>
   	<TabItem label="Windows">
   		```powershell
   		# 管理者権限で実行:
   		waired claude enable
   		```
   		インストーラーの「Route Claude Code through Waired」タスクが、これを
   		デフォルトで行ってくれます。
   	</TabItem>
   </Tabs>

3. あとは Claude Code を起動するだけです — `ANTHROPIC_BASE_URL` もエイリアスも不要です:

   ```sh
   claude
   ```
</Steps>

<Aside type="caution" title="Waired は Claude を決して壊しません">
	ローカルでの配信が一時停止・無効化・利用不可の場合、リクエストは本物の
	Anthropic API へ **fail open** (フェイルオープン) するため、Claude Code は
	動作し続けます。マネージド設定には資格情報が書き込まれないため、claude.ai
	サブスクリプションとその auto モードは維持されます。自分のモデルがいつ使われるかは、
	あなたが制御できます。

	リクエストがローカルで処理できない場合(ローカルモデルが未対応の形式や、一時的な
	エラー)、Waired は本物の Anthropic API に retry するのでターンは完了します。最後に
	これが起きた時刻は `waired claude status` で確認できます。
</Aside>

<Aside type="note" title="ローカルではどのモデルが答えるか">
	Claude Code は Anthropic のモデル名(Opus / Sonnet / Fable …)を表示・送信し
	続けます。リクエストがローカルで処理されるときは、セッションが選択している
	モデルに関わらず、**このデバイスのアクティブなローカルモデル**(`waired status`
	に表示されるモデル)が応答します。Anthropic のモデル名が意味を持つのは、
	リクエストが本物の API に送られる場合だけです。
</Aside>

<Aside type="note" title="ローカルモデルのコンテキスト窓">
	ローカルモデルのコンテキスト窓は、Claude のホスト型モデルより小さいことが
	普通です。Waired はアクティブなローカルモデルの実際の窓を Claude Code に伝える
	ので、Claude Code は会話を自動でコンパクト(要約)して窓に収め続けます —
	長いセッションでも、コンテキストの先頭が無言で切り捨てられることなく機能し
	続けます。1 ターンが単体で大きすぎて収まらない場合も、Claude Code が自動で
	コンパクトして再送します。
</Aside>

### セッション中に送信先を切り替える

送信先を変えるのに Claude Code の再起動は不要です。連携を有効化すると
**`/waired-route` スラッシュコマンド**がインストールされ、Claude Code の中で
実行すると*次の*リクエストから反映されます:

- `/waired-route auto` — 既定:Waired 優先、上記の実 Anthropic API への自動フォールバック付き。
- `/waired-route waired` — Waired 推論を強制;Anthropic には一切繋がない。
- `/waired-route anthropic` — 本物の Anthropic API を使う(ローカルモデルが特定の
  タスクでうまく動かない時に便利)。

これは**メイン会話**を設定します。サブエージェントは別途設定しない限りこれに追従します。
同じ切り替えはターミナルからも可能で、`--subagents` はバルクなサブエージェントを独立に振り分けます:

```sh
waired claude route auto                          # メイン会話
waired claude route anthropic --subagents waired  # main は Anthropic、サブは Waired
waired claude route                               # 現在のポリシーを表示
```

トレイからも変更できます: **Claude Code** メニューに **メイン会話** と(別途)
**サブエージェント**(既定は *メインと同じ*)のルート選択があります。現在の選択には
印が付き、変更は次のリクエストから反映されます — 再起動は不要です。

<Aside type="tip" title="リクエストを厳密に Waired に留める">
	ローカルで処理できないリクエストが黙って Anthropic に流れるのを避けたい場合は、
	`waired` ルートを選びます — Waired 推論のみを使い、redirect せずローカルのエラーを
	そのまま見せます:

	```sh
	waired claude route waired               # メイン会話、Anthropic に繋がない
	waired claude route --subagents waired   # サブエージェントのみ
	```
</Aside>

<Aside type="note" title="どの Waired マシン?">
	`waired` はあなたの Waired 推論を指します — *どの*マシン(この端末か mesh peer)で
	配信するかはここではなく `waired worker` で選びます。
</Aside>

<Aside type="tip" title="Web コンソールでマシンを調整する">
	[Web コンソール](https://app.waired.ai/)でマシンのページを開くと、mesh が
	Claude Code でそのマシンをどう使うかを調整できます:

	- **メイン / サブエージェントを配信** — 2 つのトグルで、mesh がメイン会話および
	  サブエージェントの処理をこのマシンへ振り分けてよいかを決めます。遅いマシンを
	  並列サブエージェント作業専用にしたいときはメインをオフにします。(これは
	  *他のマシンから*振り分けられる要求にのみ影響し、そのマシン自身のローカル
	  Claude Code 利用には影響しません。)
	- **最大同時リクエスト数** — マシンが同時に処理する要求数。増やすと多くのピアを
	  並列に手伝えますが、並列要求ごとに VRAM を消費し各要求が遅くなるため、ページは
	  推奨上限を表示し、超過保存の前に警告します。
</Aside>

### Claude Code の中で送信先を確認する

Waired が有効かどうかを迷わずに済むよう、連携を有効化すると **Claude Code の中に**
見える形で次の 2 つが追加されます:

- フッターの **ステータスライン** に現在のルートを表示 — 例:
  `⚡ waired: on Waired (qwen3-8b-instruct)`(Waired ネットワークで配信。直前の
  リクエストに応答したモデルを併記)、`→ waired: Anthropic`(本物の API を使用)、
  `✕ waired: agent down`。独自のステータスラインが無ければ Waired が追加し、
  既にある場合は `waired init` / `waired claude enable` が編集前に確認します
  (自分で `waired claude statusline install --wrap` を実行してもよい)。Waired を後で
  削除すればきれいに消えます。

  注意点がひとつ: Claude Code は**最優先スコープの**ステータスラインだけを使い、
  プロジェクトの `.claude/settings.local.json` / `.claude/settings.json` は
  Waired が入れるユーザーレベルの設定より優先されます。Waired はプロジェクトの
  ファイルを編集しません — プロジェクト内で `waired claude status` を実行すると
  shadow されていることと、自分のステータスラインスクリプトに貼り付けて
  ルートを表示できる 1 行スニペットが表示されます。
- 本物の Anthropic API に **フォールバックしたターンの後**に出る 1 行の通知 —
  ローカルで処理できなかったリクエストが黙って redirect されることはありません。

どちらも `waired claude disable` で削除されます。有効化時にステータスラインを
入れたくない場合は `--no-statusline` を付けます。

## OpenCode

OpenCode は同じ `waired link` ステップで設定されます。これにより、プロバイダー
プラグインが `~/.config/opencode/plugin/waired.js` にインストールされます。
プラグインは OpenCode を Local Gateway に向け、ゲートウェイトークンを代理で
転送します。

```sh
waired link opencode
```

エージェント名を指定せずに `waired link` を実行すると、インストール済みの全エージェントを
一度にセットアップします。

## 確認と削除

ルーティングが有効かどうかを確認します:

```sh
waired claude status
```

マネージド設定が有効かどうかと、`managed-settings.json` ファイルの場所が
表示されます:

- **Linux:** `/etc/claude-code/managed-settings.json`
- **macOS:** `/Library/Application Support/ClaudeCode/managed-settings.json`
- **Windows:** `C:\Program Files\ClaudeCode\managed-settings.json`

トレイの **Claude integration** メニューでも同じステータスが表示されます。

連携を削除するには (外科的な操作で — `link` が追加したものだけを取り消します):

```sh
waired unlink            # すべてのエージェントを削除
waired unlink claude-code  # 1 つだけを削除
```

Claude Code を Waired 経由でルーティングするのをやめるには (マネージド設定の変更を
元に戻し、Claude Code を既定の動作に戻します):

```sh
sudo waired claude disable   # Windows: 管理者権限で実行 (sudo は不要)
```

コーディングエージェントが自分のモデルを使っていない場合は、[`waired doctor`](/ja/troubleshooting/) を
実行してください。これはゲートウェイトークン、一時停止状態、エンジンの準備状況、
メッシュピア、連携を一度にチェックし、修正可能なものは修復できます。
