Waired は自分のパソコン上で、OpenAI や Anthropic の API と同じ言葉を話すアドレスで応答します。
それらに接続できるもの — ブラウザのチャット UI、エディタ拡張、スクリプト — なら、
そのまま自分の AI を使えます。

[Claude Code](/ja/guides/claude-code/)・[OpenCode](/ja/guides/opencode/)・
[OpenClaw](/ja/guides/openclaw/) は自動で設定されるので、このページは不要です。

## 設定に入れる値

<Steps>

1. **アドレス（ベース URL）** — ツールがどちらの API を話すかで変わります。

	| ツールが話す API | ベース URL |
	|---|---|
	| OpenAI | `http://127.0.0.1:9473/v1` |
	| Anthropic | `http://127.0.0.1:9473/anthropic` |

2. **モデル** — 具体的なモデル名ではなく Waired の名前を使ってください。
	モデルを切り替えても設定を変えずに済みます。

	| 名前 | 内容 |
	|---|---|
	| `waired/default` | 既定のモデル |
	| `waired/coding` | コーディング向けモデル |
	| `waired/small` | 速く短い応答 |

3. **API キー** — ツール側が必ず要求してきます。Waired が使う鍵は
	自分のパソコンの中にあります。

	<Tabs syncKey="os">
		<TabItem label="Linux">
			```sh
			cat ~/.config/waired/secrets/gateway-token
			```
		</TabItem>
		<TabItem label="macOS">
			```sh
			cat ~/Library/Application\ Support/waired/secrets/gateway-token
			```
		</TabItem>
		<TabItem label="Windows">
			```powershell
			Get-Content "$env:AppData\waired\secrets\gateway-token"
			```
		</TabItem>
	</Tabs>

	表示された値を API キーとして貼り付けます。

</Steps>

<Aside type="caution" title="このファイルはパスワードです">
	これを持っているものは自分の AI を使えます。自分だけが読める権限になっているので、
	そのままにしてください。バグ報告やスクリーンショットに貼らないでください。
</Aside>

<Aside type="note" title="鍵のファイルがまだない場合">
	コーディングツールを最初に接続したときに作られます。`waired link` を実行して作るか、
	[`waired doctor`](/ja/getting-started/doctor/) を実行してください。
	後者は実際のパスを表示し、**f** を押せば作成もします。
</Aside>

## 設定する前に確認する

```sh
curl http://127.0.0.1:9473/v1/models \
  -H "Authorization: Bearer $(cat ~/.config/waired/secrets/gateway-token)"
```

<Expected>
	`waired/default` を含む JSON の一覧が返ります。
	これがそのままチャットアプリで使えるモデル名です。
</Expected>

<Expected tone="bad">
```
curl: (7) Failed to connect to 127.0.0.1 port 9473
```
	Waired が動いていないか、ルーティングが一時停止しています。
	[`waired doctor`](/ja/getting-started/doctor/) を実行してください。
</Expected>

実際のリクエストはこうなります。

```sh
curl http://127.0.0.1:9473/v1/chat/completions \
  -H "Authorization: Bearer $(cat ~/.config/waired/secrets/gateway-token)" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "waired/default",
    "messages": [{"role": "user", "content": "Say hi"}]
  }'
```

## アプリは同じパソコンで動かしてください

`127.0.0.1` は「このパソコンだけ」という意味です。ネットワークからは意図的に
到達できないようにしてあり、それが自分の AI をうっかり公開してしまわないための仕組みです。

つまりブラウザのチャットアプリは、Waired と**同じパソコン**で動かす必要があります。
別のマシンから自分の AI を使いたい場合は、そちらにも Waired を入れて
[同じアカウントでサインイン](/ja/getting-started/add-a-device/)してください。
2 台がプライベートにつながるので、チャットアプリはそのマシンの `127.0.0.1` と話せばよくなります。

<Aside type="caution" title="手動設定はまだ手間がかかります">
	Claude Code・OpenCode・OpenClaw は自動で設定されます。それ以外も上記の設定で動きますが、
	鍵のコピーは自分で行う必要があります。アドレスとモデル名は安定しているので、
	いま設定したものはそのまま使い続けられます。
</Aside>

## エンドポイント一覧

| API | エンドポイント |
|---|---|
| OpenAI chat | `POST /v1/chat/completions` |
| OpenAI models | `GET /v1/models` |
| OpenAI responses | `POST /v1/responses` |
| Anthropic messages | `POST /anthropic/v1/messages` |

ほかのモデル名や、Waired の名前が実際にどのモデルに解決されるかは
[モデルカタログ](/ja/reference/model-catalog/)にあります。
