## 使うかどうかを決める

### セットアップは大変ですか？

インストールはコマンド 1 つです。そのあとはブラウザで進めるか、ターミナルで 2 問に
答えるかを選べます。どちらでも 10 分ほど（＋モデルのダウンロード時間）です。
→ [クイックスタート](/ja/quickstart/)

### お金はかかりますか？

かかりません。サブスクリプションも従量課金もありません。モデルはすでに持っている
ハードウェアで動くので、費用は電気代だけです。

### グラフィックボードは必要ですか？

必須ではありませんが、あると大きく違います。最近の CPU でも小さいモデルなら実用に
なりますし、グラフィックボードがあれば数倍速くなります。各モデルの要件は
[モデルカタログ](/ja/reference/model-catalog/)にありますが、セットアップが収まるものを
選ぶので読まなくても大丈夫です。

### どのツールで使えますか？

Claude Code と OpenCode はそのまま使えます。OpenAI 互換・Anthropic 互換の
クライアントなら接続できます → [チャットアプリから使う](/ja/guides/chat-clients/)

### オープンソースですか？

クライアント（自分のマシンで動くものすべて）はオープンソースで、
[GitHub](https://github.com/waired-ai/waired) で読めます。端末どうしを引き合わせる
調整サービスは、こちらでホストしている部分です。

## ハードウェアとモデル

### どの AI モデルが使えますか？

Waired はコーディング向けモデルのカタログを同梱し、そのマシンで実際に動かせる中から
最良のものを選びます。あとから変更できます
→ [使う AI モデルを選ぶ](/ja/guides/choose-a-model/)

### モデルはどうやって選ばれるのですか？

CPU・メモリ・グラフィックボードを見て、余裕をもって収まる中で最も品質の高いものを
選びます。そのあと実速度を測り、追いつかない場合はより軽いものを提案します。
ディスクを埋めることはありません — 空き容量が足りなければ、途中で失敗する代わりに
小さいモデルに落とします。

### 推奨より大きいモデルも動かせますか？

はい。警告して不足分を表示しますが、禁止はしません。少しの超過ならたいてい動き、
単に遅くなります。本当に大きすぎる場合は読み込みに失敗します
→ [ハードウェアより大きいモデルを選んでしまった](/ja/troubleshooting/#i-chose-a-model-bigger-than-my-hardware)

## プライバシーとネットワーク

### プライバシーは守られますか？

入力と応答は、自分の端末どうしをエンドツーエンド暗号化された経路で行き来します。
調整サービスは端末どうしを引き合わせるだけで内容を受け取らず、直接つながらないときだけ
使われる中継サーバーは、読めない状態のデータをそのまま転送します。詳細は
[プライバシー](/ja/concepts/privacy/)。

### 自分のモデルが止まっていたら、クラウドに送られますか？

Claude Code に限り、自分の側で応答できない場合だけ、そして**黙って送られることは
ありません**。ターンを完了させるために本来の Anthropic API が使われ、そのことが
必ず表示されます。エラーを見たい場合は `waired` ルートを選んでください
→ [Claude Code から使う](/ja/guides/claude-code/)

### ポート開放や VPN の設定は必要ですか？

不要です。ネットワークが許せば端末どうしが直接つながり、ファイアウォールや厳しい NAT が
邪魔する場合は暗号化された中継に切り替わります。どちらも自動です。

### サインインの仕組みは？

Google でサインインします。**同じアカウント**でサインインしたパソコンどうしが同じ
プライベートネットワークに入り、互いに到達できるようになります。これが仕組みのすべてで、
ペアリング操作もアドレスの入力もありません。

### オフラインでも使えますか？

モデルをダウンロードし終えていれば、そのパソコンはインターネットなしで応答できます。
*ほかの端末から*そのパソコンに届くには両者の間にネットワーク経路が必要ですが、
同じ家庭内・社内ネットワークならオフラインでも動きます。

## 運用する

### アップデートはどうしますか？

`waired update`、または更新があるときに Waired アイコンに出る項目から実行します
→ [アップデート](/ja/getting-started/update/)

### 削除するには？

コマンド 1 つ・約 10 秒です。ダウンロード済みモデルを残すかどうかも選べます
→ [アンインストール](/ja/getting-started/uninstall/)

### 調子が悪いとき、まず何をすればいいですか？

`waired doctor` です。全体を点検し、直せるものは修復します。そのあとは症状別に
まとめた[うまくいかないとき](/ja/troubleshooting/)へ。
