## セットアップは難しいですか？

いいえ。短いコマンドを 1 つ実行して Google でサインインすれば、コーディング
エージェントが自動的にリンクされます。ネットワークの知識は一切不要です。
[インストール](/ja/getting-started/install/) を参照してください。

## プライベートですか？

はい。あなたのプロンプトと返信は、エンドツーエンドで暗号化されたリンク上を、
あなた自身のマシン同士の間で直接やり取りされます。当プロジェクトのサーバーに届くことは決して
なく、リレーでさえそれらを読むことはできません。
[プライバシー](/ja/concepts/privacy/) を参照してください。自分のデバイスを超えた
共有 — チームや [パブリック共有](/ja/public-share/) — はオプトインで、同意を
経たものだけです。その場合でも、何を尋ね、何が答えられたかを Waired のサーバーが
見ることは決してありません。

## 何が必要ですか？

モデルを動かすコンピュータ（GPU があると助かりますが、最近の CPU でも 7B モデルは
問題なく動きます）と、普段使いのノートパソコンです。あなたが入力するクライアント
マシンには、特別なハードウェアは必要ありません。

## お金はかかりますか？

ソフトウェアは無料で、モデルはあなたがすでに所有しているハードウェア上で動作するため、
メッセージごとの課金もサブスクリプションもありません。

## どの AI モデルが動作しますか？

**Ollama** で動かせる任意のローカルモデルに加え、NVIDIA/AMD GPU サーバー向けには
**vLLM** でより大きなモデルが動作します。Waired はデフォルトでコーディングモデルを
バンドルしており、いつでも別のモデルに差し替えられます —
[モデルカタログ](/ja/reference/model-catalog/) と
[バンドルモデルを切り替える](/ja/guides/switch-model/) を参照してください。

## どのツールが動作しますか？

Claude Code と OpenCode は `waired link` によって自動的にセットアップされます。その他の
OpenAI 互換または Anthropic 互換のクライアントも、[ローカルゲートウェイ](/ja/guides/chat-clients/)
に向けることで動作します。

## オープンソースですか？

Waired の **クライアント** はオープンソースです。あなたのマシン上で何が動いているかは
[GitHub](https://github.com/waired-ai/waired) で正確に読むことができます。あなたのデバイスを
引き合わせるコーディネーションサービスが、あなたのためにホストされている部分です。

## サインインはどのように機能しますか？

`waired init` で Google にサインインし、デバイスをエンロールします。すべてのデバイスを
**同じ Google アカウント** でエンロールしてください。その共有された ID が、デバイスを
同じプライベートネットワークに置き、あるデバイスが別のデバイスのモデルを使えるように
するものです。アカウントの外の人との共有 — チームやパブリック共有 — は、別途の
明示的なオプトインです。

## 他の人が私のコンピューターを使えますか？

[パブリック共有](/ja/public-share/) をオンにした場合（またはチームに参加して
共有した場合）だけです。デフォルトはオフです。あなたが明示的に有効にするまで、
あなたのアカウント外の誰もあなたのコンピューターで作業を実行することはできず、
再びオフにすれば即座に効きます。

## ネットワークに厳格なファイアウォールや NAT がある場合はどうなりますか？

デバイスはまず直接 UDP リンクを試みます。厳格な NAT やファイアウォールがそれを
ブロックする場合、*暗号化された* トラフィックを転送するリレーへ自動的に
フォールバックします。設定は何も必要なく、どちらの場合でも接続性は機能します。
（Waired はポートを開けるために UPnP や NAT-PMP を使用しません。）

## セットアップ後にオフラインで使えますか？

入力しているのと同じマシンでモデルを動かす場合はローカルで機能します。*リモート* な
デバイスのモデルを使うにはネットワーク接続が必要です。なぜなら、ピアを発見し、署名済み
ネットワークマップを最新に保つのがコントロールプレーンだからです。コントロール
プレーンがあなたのプロンプトを見ることは決してありません — 鍵とエンドポイントを
配るだけです。

## Waired はどのようにモデルを選びますか？

Auto-Selector は、お使いのハードウェアのメモリに収まる中で最も高品質なモデルを選び、
共有メモリマシンでは効率的な Mixture-of-Experts モデルを優先します。
`waired infer --explain "say hi"` でルーティングの判断をプレビューできます。

## ローカルモデルがダウンしたらどうなりますか？

Claude Code については、[マネージド設定による連携](/ja/guides/coding-agents/) が
**フェイルオープン** です。ローカルのサービングが一時停止・無効化・利用不可の場合、
リクエストは実際の Anthropic API へ送られ、Claude は動作し続けます。そしてルーティングの
状態は `waired claude status` と `waired doctor` で常に可視です。マネージド設定には
資格情報が書き込まれないため、claude.ai サブスクリプションは終始そのまま使われます。
Waired が静かにあなたのコーディングエージェントを壊すことは決してありません。
