Waired は 1 つのコマンドでインストールできます。インストーラーはクライアント
（`waired` CLI + `waired-agent` バックグラウンドサービス）を配置し、自動起動するように
登録し、さらに — ほとんどのプラットフォームでは — [Ollama](https://ollama.com)
推論エンジンも合わせてインストールします。

## 対応プラットフォーム

| 役割 | 対応システム |
|---|---|
| 推論サーバー（モデルを実行する） | Ubuntu 24.04+、Debian trixie+、Windows 11 / Server 2025、macOS 13+（Apple Silicon 推奨） |
| クライアント（普段使いのノートパソコン） | 上記のいずれか — GPU は**不要**です |

GPU があれば推論は高速になりますが、最近の CPU でも 7B モデルは問題なく動きます。
入力に使うクライアントマシンには特別なハードウェアは必要ありません。

## インストール

<Tabs syncKey="os">
	<TabItem label="Linux">
		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh
		```

		`waired` apt パッケージと systemd ユニットをインストールし、Ollama も
		自動でインストールします。apt リポジトリは **Debian trixie+ / Ubuntu 24.04+
		のみ**を対象としています。その他のディストリビューションは、現時点ではワンライナーでは
		サポートされていません。
	</TabItem>
	<TabItem label="macOS">
		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh
		```

		同じスクリプトが macOS を検出し、クライアントを `/usr/local/bin` に
		インストールし、`waired-agent` をシステム launchd デーモンとして登録し、
		Ollama をインストールします。エージェントはログイン不要でブート時に起動します
		（Linux / Windows のサービスと同様）。そのため特権ステップで `sudo` を使い、
		管理者パスワードの入力を求められます。バイナリは現時点では署名されていません
		（ad-hoc）。署名と Homebrew formula は計画中です。
	</TabItem>
	<TabItem label="Windows">
		```powershell
		iwr -useb https://github.com/waired-ai/waired-agent/releases/latest/download/install.ps1 | iex
		```

		クライアントをインストールし、`waired-agent` Windows サービスを登録し、
		Ollama をインストールします（昇格ステップは UAC プロンプトが処理します）。
		エージェントはログインとは独立してブート時に起動します — Linux / macOS の
		インストーラーと同じ既定動作です。約 1.4 GB の Ollama ダウンロードは進捗を
		定期的に表示します。

		<Aside type="note" title="Windows の Ollama">
			同梱エンジンを**インストールしない**場合（ヘッドレスやディスク容量の
			少ないホストなど）は、`-SkipOllama` を渡すか `WAIRED_NO_OLLAMA=1` を
			設定します（後述の「同梱の Ollama をスキップする」を参照）。あとから
			昇格プロンプトで `waired runtimes install ollama` を実行して追加できます。
		</Aside>
	</TabItem>
</Tabs>

どの OS でもインストーラーはまず、これから行うことの要約（インストール先、
バックグラウンドサービス、Ollama のダウンロード、ブラウザでのサインイン、管理者権限の
プロンプト）を表示し、`Proceed? [Y/n]` の確認を取ってから実行を始めます。無人ホストでは
`--yes` / `-Yes` で確認を省略できます。Windows では対話的な新規インストール時に
インストール先も選べます（Enter で既定の `C:\Program Files\Waired`。
`-InstallDir <path>` または `WAIRED_INSTALL_DIR` で固定可能）。

インストーラーが完了したら、[初回起動](/ja/getting-started/first-run/)に進んで
デバイスを登録してください。

<Aside type="note" title="GNOME のシステムトレイ（Linux）">
	GNOME には組み込みのシステムトレイが無いため、トレイアイコンの表示には
	AppIndicator ホスト拡張が必要です。`waired init` は GNOME を検出すると拡張を
	自動でインストール・有効化します（KDE Plasma はそのまま動作します）。それでも
	アイコンが表示されない場合は [トラブルシューティング](/ja/troubleshooting/) を
	参照してください。
</Aside>

### 同梱の Ollama をスキップする

すでに Ollama を動かしている場合や、自分でインストールしたい場合は、フラグまたは
環境変数のいずれかで同梱エンジンを抑制できます。

<Tabs syncKey="os">
	<TabItem label="Linux / macOS">
		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh -s -- --skip-ollama
		# または:
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | WAIRED_NO_OLLAMA=1 sh
		```
	</TabItem>
	<TabItem label="Windows">
		```powershell
		# スクリプトをファイルに保存した状態で:
		.\install.ps1 -SkipOllama
		# または実行前に環境変数を設定する:
		$env:WAIRED_NO_OLLAMA = '1'
		```
	</TabItem>
</Tabs>

### デフォルト以外の構成

特定の（セルフホストやステージングなどの）コントロールプレーンに向けたインストール、
バージョンの固定、その他のインストーラーフラグは
[高度なインストールオプション](/ja/reference/install-options/)にまとめています —
通常のインストールではいずれも不要です。

<a id="update"></a>
## アップデート

インストール済みホストをアップデートする最も簡単な方法は CLI です（またはトレイの
**「⚠ Update available」**バナー）。

```sh
waired update          # チェックし、より新しいバージョンがあれば適用する
waired update --check  # レポートのみ（読み取り専用）
waired update --yes    # 対話的プロンプトなしで適用する（非対話的ホスト向け）
waired update --edge   # edge チャンネルで更新／edge に切り替え
waired update --stable # stable チャンネルで更新／stable に切り替え
```

`waired update` は現在追従しているチャンネルを維持します — stable 版は最新 stable へ、
edge 版は最新 edge へ更新され、edge ホストが黙って stable に落ちることはありません。
チャンネルを切り替えるには `--edge` / `--stable` を使います（下の
「edge チャンネル」の節を参照）。

バックグラウンドサービスも数時間ごとにアップデートをチェックして通知します。適用は
常にユーザーが起動する操作です。アップデートを行っても、登録情報、アイデンティティ、
ディスク上の状態は保持されます。同梱の Ollama は別途管理されており、`waired update`
では変更されません。

## アンインストール

対応するワンライナーで Waired クライアントを削除します。バイナリの削除とバックグラウンド
サービスの登録解除に加えて、ベストエフォートで**このデバイスを Waired アカウントから
登録解除します**（デバイス一覧から消え、ネットワークからも外れます）。ローカルの設定・
状態はデフォルトでは**保持されます**（下記の `--clean` で併せて削除可能）。いずれの場合も、
再インストール時は `waired init` で改めてサインインします。

<Aside type="note" title="アンインストール時にオフラインだった場合">
	登録解除はベストエフォートであり、アンインストールを妨げることはありません。その時点で
	デバイスが Waired に到達できない場合（オフライン、またはサービス停止済みなど）は、
	デバイス一覧に残ることがあります。その場合は Web 管理画面から手動で削除してください。
</Aside>

<Tabs syncKey="os">
	<TabItem label="Linux / macOS">
		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/uninstall.sh | sh
		```
	</TabItem>
	<TabItem label="Windows">
		```powershell
		iwr -useb https://github.com/waired-ai/waired-agent/releases/latest/download/uninstall.ps1 | iex
		```
	</TabItem>
</Tabs>

アンインストーラーは削除内容の一覧を表示し、`Proceed? [y/N]`（Enter で中止）の確認を
取ってから実行します。確認を省略するには `--yes` / `-Yes`、変更前にプレビューするには
`--dry-run` / `-DryRun` を付けます。Windows では昇格ウィンドウが最後に
「Press Enter to close」で停止するので、成否を必ず確認できます。

**完全に削除する**場合 — 設定や状態に加えて、同梱の Ollama とダウンロード済みモデルも
削除します — `--clean`（Linux / macOS）または `-Clean`（Windows）を付けます。これは
破壊的な操作で、非対話的なシェルではさらに `--yes` / `-Yes` の明示が必須です。

<Tabs syncKey="os">
	<TabItem label="Linux / macOS">
		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/uninstall.sh | sh -s -- --clean
		```
	</TabItem>
	<TabItem label="Windows">
		```powershell
		# -Clean はスクリプトをファイルに保存して実行する必要があります（パイプ形式ではフラグを渡せません）:
		iwr -useb https://github.com/waired-ai/waired-agent/releases/latest/download/uninstall.ps1 -OutFile $env:TEMP\uninstall.ps1
		& $env:TEMP\uninstall.ps1 -Clean
		```
	</TabItem>
</Tabs>

<Aside type="note" title="Windows の GUI インストーラーで入れた場合">
	`WairedSetup-<version>-x64.exe` で Waired をインストールした場合は、**設定 → アプリ
	→ Waired → アンインストール**からも削除できます。上記のワンライナーはどちらの方法で
	インストールした場合でも安全に実行できます。
</Aside>

<a id="clean-install"></a>
## クリーンインストール（完全削除してから再インストール）

ホストをまっさらな状態から作り直すには、*インストーラー*にクリーンオプションを
付けて実行します。1 コマンドで上記の完全削除アンインストール（設定・鍵・状態・
同梱の Ollama とそのモデルをすべて削除）を行ってから、新規にインストールします。
削除の前に確認を求めます — 非対話的なホストでは `--yes` / `-Yes` を付けて
プロンプトをスキップしてください。

<Tabs syncKey="os">
	<TabItem label="Linux / macOS">
		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh | sh -s -- --clean
		```
	</TabItem>
	<TabItem label="Windows">
		```powershell
		# パイプ形式では -Clean を渡せないため、環境変数でオプトインします:
		$env:WAIRED_CLEAN = '1'
		iwr -useb https://github.com/waired-ai/waired-agent/releases/latest/download/install.ps1 | iex
		```

		UAC プロンプトが **2 回**表示されます — 1 回目は削除、2 回目はインストールです。
	</TabItem>
</Tabs>

削除によってこのデバイスの古いアイデンティティは消えるため、インストール終了時の
サインイン（[初回起動](/ja/getting-started/first-run/)）で新しいデバイスとして
登録されます。

<a id="edge"></a>
## edge チャンネル（最新・不安定版）

安定リリースを待たずに最新の `main` ビルドを追いかけるには、**edge** チャンネルに
オプトインします。edge はマージのたびに再ビルドされ、**本番利用向けではありません**。

<Tabs syncKey="os">
	<TabItem label="Linux / macOS">
		```sh
		curl -fsSL https://github.com/waired-ai/waired-agent/releases/latest/download/install.sh \
		  | WAIRED_VERSION=edge sh
		```
	</TabItem>
	<TabItem label="Windows">
		```powershell
		$env:WAIRED_VERSION = 'edge'
		iwr -useb https://github.com/waired-ai/waired-agent/releases/latest/download/install.ps1 | iex
		```
	</TabItem>
</Tabs>

edge に切り替えた後は `waired update` が自動的に edge を維持します。後からチャンネルを
切り替えるには `waired update --edge` または `waired update --stable` を実行します
（インストーラーを `--edge` / `--stable` 付きで再実行しても同じ）— apt ソース（Linux）
またはリリースアセットを差し替え、どちらの方向のバージョン変更も許可します。
