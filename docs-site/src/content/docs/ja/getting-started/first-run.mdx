ネットワークに入れたいパソコンごとに、1 回ずつ実行します。

<Aside type="tip" title="同じアカウント = 同じネットワーク">
	**すべての**パソコンで**同じ Google アカウント**を使ってください。この共通の
	サインインが、あるマシンの AI を別のマシンから使えるようにしています。
	別々のアカウントだと、互いにつながらない別のネットワークになります。
</Aside>

## セットアップを開始する

<Tabs syncKey="os">
	<TabItem label="Windows">
		**管理者として PowerShell を開き**（スタートボタンを右クリック →
		「ターミナル（管理者）」）、次を実行します。

		```powershell
		& "C:\Program Files\Waired\waired.exe" init
		```
	</TabItem>
	<TabItem label="macOS">
		```sh
		sudo waired init
		```
	</TabItem>
	<TabItem label="Linux">
		```sh
		sudo waired init
		```
	</TabItem>
</Tabs>

サインイン用のリンクが表示され、**Enter** を押すとブラウザーで開きます。
リンクを自分でコピーしてブラウザーに貼っても構いません。

<Aside type="caution" title="セットアップが終わるまで、このウィンドウは閉じないでください">
	セットアップはブラウザに移りますが、管理者権限が必要な作業（AI ソフトの導入と
	コーディングツールの連携）はこのウィンドウが担当し続けます。

	閉じてしまった場合はブラウザにそう表示されます
	（*「The setup command on … was closed before this finished. Your progress was saved.」*）。
	同じコマンドをもう一度実行すれば続きから再開できます。進捗は失われず、
	モデルのダウンロードはそのまま続きます。
</Aside>

## ブラウザで完了させる

サインインすると、いま設定中のパソコンのセットアップ画面に移動します。
そのパソコンの状況がそのまま表示されます。

<Screenshot
	src="setup-wizard-progress.png"
	alt="ブラウザに表示された Waired のセットアップ画面。Install the AI software（完了）、Download the AI model（19.2 GB 中 7.9 GB で進行中）、Check the speed（待機中）の 3 ステップ。"
	caption="パソコン自身が報告している 3 つの工程です。このタブを閉じても止まりません。"
/>

| 工程 | 何をしているか | 目安 |
|---|---|---|
| **Install the AI software** | モデルを読み込んで動かすプログラムの導入。管理者権限が必要なため、ターミナルを開いたままにする必要があります。 | 1〜2 分 |
| **Download the AI model** | AI 本体。このパソコンで実際に動かせる最良のモデルを Waired が選びます。 | 5〜30 分 |
| **Check the speed** | 速度の計測。このマシンが追いつけるかを判断します。 | 最大 5 分 |

続いて、どのコーディングツールを連携するか聞かれます。使っているなら
**Claude Code** を有効にしてください（あとから変更できます）。

完了すると、そのデバイスのページに着地します。以降の確認・変更はそこから行えます。

<Aside type="note" title="セットアップ画面は英語表示です">
	Web 画面は現在英語のみです。日本語化は予定されています。
</Aside>

## ここで決めていること

2 つの設定が、このパソコンの役割を決めます。ハードウェアから妥当な既定値が選ばれ、
どちらもあとから変更できます。

**このパソコンで AI モデルを動かすか**
: デスクトップや GPU マシンなら「はい」。ほかのパソコンの AI を*使うだけ*の
  ノートパソコンなら「いいえ」— ネットワークには参加しますが、モデルは持ちません。
  実用的なモデルが載らないマシンでは既定が「いいえ」になり、何が足りないかが表示されます。

**ほかの端末からこのパソコンの AI を使えるようにするか**
: ノートパソコンやスマートフォンに応答させたいマシンなら「はい」。「いいえ」にすると
  このパソコン専用になります（自分では引き続き使えます）。1 つ目の設定とは独立です
  — [共有と一時停止の違い](/ja/reference/cli/#sharing-vs-pausing)を参照してください。

<Aside type="caution" title="「非常に小さいモデルしか動かせません」と出たら">
	メモリの少ないパソコンでは、そう報告したうえで「動かさない」ことを勧められます。
	その助言に従ってください。そのサイズのコーディングモデルは、役に立つ出力より
	壊れた出力のほうが多くなります。それでもこのパソコンをネットワークに入れる価値は
	あります — ほかのパソコンの AI を使えるからです。
</Aside>

## ターミナルだけで進める

ブラウザでの操作は便利のためであって、必須ではありません。開くブラウザがない場合
（SSH 越しのサーバー、デスクトップ環境のないマシン、`--no-browser`、
`--non-interactive`）は自動的にターミナル方式になり、次の表示で **Enter** を押せば
いつでも切り替えられます。

```
Setup is continuing in your browser…
(press Enter to continue in the terminal instead)
```

聞かれる内容は同じです。

<Steps>

1. **Google サインイン。** リンクが表示されるので Enter で開きます。ディスプレイの
	ないマシンでは URL とペアリングコードだけが表示されるので、別の端末で開きます。

2. **Run AI models on this computer? (Y/n)** — 上記 1 つ目の設定です。既定値は
	このマシンのメモリから決まります。

3. **Let your other devices use this computer's AI? (Y/n)** — 2 つ目の設定です。

</Steps>

モデルを有効にした場合は、収まるモデルを選び、ディスク空き容量を確認したうえで
（足りなければディスクを埋めるのではなく小さいモデルに落とします）、
進捗表示つきでダウンロードします。

```
Starting the inference engine…
Preparing to download qwen3-coder-30b-a3b-instruct…
Downloading qwen3-coder-30b-a3b-instruct: 41%  7.9 GB / 19.2 GB (44 MB/s)
```

そのあと速度を計測し、追いつかない場合はより軽いモデルを提案します
（両方のモデル名と品質ランクを並べて表示するので、速さと質を見比べられます）。
最後に、Claude Code を Waired 経由にするか聞かれます。「いいえ」でも連携は
インストールされたまま無効になるだけで、あとから `sudo waired claude enable` で
有効にできます。

**こう表示されれば完了です**:

```
╭─ 🎉 Waired is ready — everything completed successfully! ─────────╮
│  Account   you@example.com                                       │
│  Device    my-desktop  d-4f2a…                                   │
│  Network   personal  overlay 100.64.0.3                          │
│  Model     qwen3-coder-30b-a3b-instruct  (58 tok/s)              │
│  Gateway   http://127.0.0.1:9479                                 │
│  Claude    http://127.0.0.1:9473 (managed settings)              │
│                                                                  │
│  Point your coding agent at Waired and start building.           │
╰──────────────────────────────────────────────────────────────────╯
```

<details>
<summary>サーバー・スクリプト向けのオプション</summary>

| フラグ | 効果 |
|---|---|
| `--no-browser` | ブラウザを開かず、URL とペアリングコードのみ表示 |
| `--non-interactive` | ハードウェアから決まる既定値を使い、何も質問しない |
| `--no-wait-model` | すぐシェルに戻り、ダウンロードは裏で続行 |
| `--inference-bundled-model-id <id>` | 自動選択ではなくモデルを指定 |
| `--inference-enabled=true` | 小さすぎると判定されたマシンでもモデルを動かす |
| `--control <url>` | 既定以外の接続先を指定 |
| `--mask-pii` | ホームディレクトリ・ユーザー名・ホスト名・メールアドレスを伏せる（不具合報告用） |

</details>

## もう一度セットアップする

すでにサインイン済みのパソコンでもう一度実行しても安全です。再認証してセットアップ
画面を開き直すだけで、**デバイス数の上限には数えられません**。
