**通常のインストールでは、これは自動的に行われます。**インストールコマンドが Waired を起動し、
そのままサインインへ進むので、初回セットアップは[クイックスタート](/ja/quickstart/)で完結し、
`waired init` を自分で打つ機会はありません。

このページは、自分で打つ必要がある場合のためのものです。

- セットアップが中断され、再開したい
- `--no-init` を付けてインストールした
- 画面のないサーバーを設定している
- それぞれの選択が実際に何を決めているのか知りたい

<Aside type="tip" title="同じアカウント = 同じネットワーク">
	**すべての**パソコンで**同じ Google アカウント**を使ってください。この共通の
	サインインが、あるマシンから別のマシンの AI を使えるようにしています。
	アカウントが違うと、互いにつながらない別々のネットワークになります。
</Aside>

## 開始する

<Tabs syncKey="os">
	<TabItem label="Windows">
		**ターミナル（管理者）**を開き（スタートボタンを右クリック）、次を実行します。

		```powershell
		waired init
		```

		<Aside type="note" title="「waired は認識されません」と出る場合">
			インストーラは Waired をシステムのパスに追加しますが、インストール**前**に
			開いていたターミナルは古いパスのままです。**新しい**管理者ターミナルを
			開いてください。開きたくない場合はフルパスで実行します:
			`& "C:\Program Files\Waired\waired.exe" init`
			（インストール先を変更した場合はそのフォルダに読み替えてください）
		</Aside>
	</TabItem>
	<TabItem label="macOS">
		```sh
		sudo waired init
		```
	</TabItem>
	<TabItem label="Linux">
		```sh
		sudo waired init
		```
	</TabItem>
</Tabs>

サインイン用のリンクを表示して待ちます。

<Expected>
```
Sign in using this link:
  https://app.waired.ai/login/…

>> Press Enter to open your browser (or open the link above yourself)...
```
</Expected>

**Enter** を押して Google アカウントを選びます。リンクを任意のブラウザに
コピーしても構いません（このパソコンでも、別のパソコンでも）。

<Aside type="caution" title="セットアップが終わるまでウィンドウを閉じないでください">
	AI ソフトウェアのインストールには管理者権限が必要で、Web ページからは取得できません。
	**このコマンドが、その作業を実際に行っています**（ブラウザ側の画面は指示を出す役です）。

	閉じてしまうと、ブラウザに *「The setup command on … was closed before this finished.
	Your progress was saved.」*と表示されます。同じコマンドを実行し直せば続きから進みます。
	失われるものはなく、モデルのダウンロードはそのまま続いています。
</Aside>

## ブラウザで終える

サインインすると、ブラウザはこのパソコンのセットアップ画面に着きます。
モデルを選んで **Yes, set it up** を押すと、そのパソコンの状況がリアルタイムで表示されます。

<Screenshot
	src="setup-wizard-progress.png"
	alt="ブラウザに表示された Waired のセットアップ画面。3 つのステップ — AI ソフトウェアのインストール（完了）、AI モデルのダウンロード（進行中、19.2 GB 中 7.9 GB のバー）、速度の確認（待機中）。"
	caption="3 つのステップは、パソコン自身が報告しています。このタブを閉じても止まりません。"
/>

| ステップ | 何をしているか | 所要時間 |
|---|---|---|
| **AI ソフトウェアのインストール** | モデルを読み込んで動かすプログラム。管理者権限が必要なのはここで、だからターミナルを開いたままにする必要があります。 | 1〜2 分 |
| **AI モデルのダウンロード** | AI 本体。数 GB あります。 | 5〜30 分 |
| **速度の確認** | このマシンが実用的な速度を出せるかの短い計測。 | 最大 5 分 |

<Expected>
	すべてのステップにチェックが付き、
	**「Everything's ready — this computer can now run AI on its own.」**と表示されます。
	**Go to device** を押します。
</Expected>

ブラウザが主導している間、ターミナルは何も聞いてきません。
コーディングツールをあとから設定できる旨を 1 行表示するだけです。

```
You can set up your coding tools later from this terminal with `waired link all`.
```

<Aside type="note" title="セットアップ画面は英語のみです">
	Web コンソールは現在英語のみです。翻訳は予定されています。
</Aside>

## ターミナルで進める

*「Setup is continuing in your browser…」*の表示で **Enter** を押すと、
ターミナル側に切り替わります。

そこで何を聞かれるかは、どういう状態で動いているかによります。

### 通常のインストール後

コーディングツールについてだけ聞かれます。モデルはブラウザ側、
またはインストーラ側ですでに決まっているためです。

<Expected title="聞かれること">
	Claude Code を Waired 経由にするかどうか。**Y** と答えます。

	いいえと答えても連携自体はインストールされ、無効なだけです。
	あとから `sudo waired claude enable` で有効にできます。
</Expected>

### <a id="no-desktop-no-browser"></a>サーバー、またはブラウザがない場合

`--no-browser`、`--non-interactive`、あるいは引き継ぎ先のサービスに到達できない環境では、
`waired init` がすべてを自分で行い、質問も一通り聞いてきます。

<Steps>

1. **Google サインイン。**リンクとペアリングコードだけが表示されるので、
	別の端末で開きます。

2. **Run AI models on this computer? `[Y/n]`** — デスクトップや GPU マシンなら **Y**。
	ほかのパソコンの AI を*使うだけ*のマシンなら **N**（それでもネットワークには参加します）。
	既定値はこのマシンのメモリ量に従います。

3. **Let your other devices use this computer's AI? `[Y/n]`** — ノートパソコンに
	提供するためのマシンなら **Y**。前の回答とは独立した設定です。

4. **Let Waired install and manage its own Ollama? `[Y/n]`** — 自前の Ollama が
	すでに入っている場合にのみ聞かれます。**n** なら自前のものをそのまま使います。

5. **Route Claude Code through Waired? `[Y/n]`**

6. **Run a quick performance benchmark now? `[Y/n]`** — モデルの準備ができてから
	聞かれます。実施をおすすめします。このパソコンの速度が足りない場合、
	軽いモデルを提案し、両方の品質ランクを並べて示すので判断できます。

</Steps>

そのあとモデルが進捗表示付きでダウンロードされます。

```
Starting the inference engine…
Preparing to download qwen3-coder-30b-a3b-instruct…
Downloading qwen3-coder-30b-a3b-instruct: 41%  7.9 GB / 19.2 GB (44 MB/s)
```

<Expected>
```
╭─ 🎉 Waired is ready — everything completed successfully! ─────────╮
│  Account   you@example.com                                       │
│  Device    my-desktop  d-4f2a…                                   │
│  Network   personal  overlay 100.64.0.3                          │
│  Model     qwen3-coder-30b-a3b-instruct  (58 tok/s)              │
│  Gateway   http://127.0.0.1:9479                                 │
│  Claude    http://127.0.0.1:9473 (managed settings)              │
│                                                                  │
│  Point your coding agent at Waired and start building.           │
╰──────────────────────────────────────────────────────────────────╯
```
</Expected>

<Aside type="caution" title="「非常に小さいモデルしか動かせません」と出たら">
	メモリの少ないパソコンでは、ごく小さいモデルしか収まらないため
	**動かさないこと**を勧められる場合があります。その助言に従ってください。
	そのサイズのコーディングモデルは、役に立つ出力より壊れた出力のほうが多くなります。
	そのパソコンをセットアップする価値はあります — ほかのマシンの AI を使えるからです。
</Aside>

<details>
<summary>サーバー・スクリプト向けのオプション</summary>

| フラグ | 効果 |
|---|---|
| `--no-browser` | ブラウザを開かず、URL とペアリングコードを表示する |
| `--non-interactive` | ハードウェアから導かれた既定値を採用し、何も聞かない |
| `--no-wait-model` | すぐシェルに戻り、ダウンロードはバックグラウンドで続ける |
| `--inference-bundled-model-id <id>` | 自動選択ではなくモデルを指定する |
| `--inference-enabled=true\|false` | 1 つめの質問に、聞かれずに答える |
| `--share-with-mesh=true\|false` | 2 つめの質問に、聞かれずに答える |
| `--control <url>` | 既定以外のサービスを指定する |
| `--mask-pii` | 出力中のホームディレクトリ・ユーザー名・ホスト名・メールアドレスを伏せる（バグ報告用） |

</details>

## Waired アプリからサインインする場合

[Waired アプリ](/ja/guides/waired-app/)には **Log in…** があり、
このパソコンをサインインさせることはできます。

<Aside type="caution" title="ただし初回セットアップは完結しません">
	AI ソフトウェアのインストールに必要な管理者権限を、アプリ側から取得できないためです。
	未セットアップのパソコンでアプリからサインインすると、セットアップ画面までは進み、
	結局ターミナルで `waired init` を実行するよう促されます。

	**Log in…** はサインアウト後の再サインインや、すでにセットアップ済みのパソコンで使ってください。
	初回は上記のコマンドを使います。
</Aside>

## もう一度実行する場合

すでにサインイン済みのパソコンでセットアップをやり直しても安全です。
再認証してセットアップ画面を開き直すだけで、**新しい端末として数えられることはありません**。
