手軽な順に 3 つあります。どれかが失敗したら、次に実行するのは `waired doctor` です。

## 1. Waired アイコンを見る

**Waired アイコン**をクリックします（macOS はメニューバー、Windows・Linux は時計のとなり）。

<Screenshot
	src="app-ready.png"
	alt="Waired のメニュー。サインイン済みのアカウント、接続状態、Inference サブメニューに動作中のモデル名が表示されている。"
	caption="サインイン済み・接続済み・モデル読み込み済み。"
	width="26rem"
/>

**正常な状態:** 先頭に自分のアカウント、接続済みの表示、そしてモデルを動かす
パソコンなら **Inference** に動作中のモデル名。

## 2. 話しかけてみる

```sh
waired infer "say hi"
```

**正常な状態:** 数秒であいさつが返ってくる。

これはこのパソコンのモデルに直接送るので、ネットワークの状態に関係なく動きます。
AI 自体が生きているかを確かめる最短の方法です。

どこで処理されたか、なぜそう判断したかを見るには:

```sh
waired infer --explain "say hi"
```

## 3. 詳細ステータスを読む

```sh
waired status --observability
```

モデルを動かすパソコンでの**正常な表示**:

```
Observability:
  Device:   my-desktop  uptime: 1m23s
  Engine:   ready (model=qwen3-coder-30b-a3b-instruct, 0/4 slots used, inflight=0)
  Share:    enabled    Paused: no
  Mesh:     2 enrolled / 1 reachable / 1 ready
  Last:     -  (no inference requests yet)
```

| 行 | 見るべき点 |
|---|---|
| **Engine** | `ready` ならモデルが読み込まれ応答可能。`not ready` は多くの場合ダウンロード中 — [応答が返ってこない](/ja/troubleshooting/#no-answer-comes-back)を参照。 |
| **Share** | `enabled` なら、ほかの端末からこのパソコンの AI を使えます。自分だけで使うなら off でも問題ありません。 |
| **Paused** | `no` なら処理中。`yes` の場合は `waired resume`。 |
| **Mesh** | パソコン全体の `enrolled / reachable / ready`。ノートパソコンでは、AI マシンが動いているかをここで確認します。 |

`-o json` を付けると機械可読な形式で出力されます。

<Aside type="tip" title="モデルを動かさないノートパソコンの場合">
	そのマシンで `Engine: not ready` と出るのは正常です（モデルを持たない役割のため）。
	大事なのは **Mesh** の行で、`1 ready` 以上あれば、応答してくれるパソコンが
	存在しているということです。
</Aside>
