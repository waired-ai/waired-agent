Waired answers on your own computer at an address that speaks the same language
as the OpenAI and Anthropic APIs. Anything that can talk to those — a browser
chat UI, an editor extension, a script — can use your AI instead.

[Claude Code](/guides/claude-code/), [OpenCode](/guides/opencode/) and
[OpenClaw](/guides/openclaw/) are set up for you and need none of this page.

## What to put in the settings

<Steps>

1. **Address (base URL)** — depends on which API your tool speaks:

	| Your tool speaks | Base URL |
	|---|---|
	| OpenAI | `http://127.0.0.1:9473/v1` |
	| Anthropic | `http://127.0.0.1:9473/anthropic` |

2. **Model** — use a Waired name rather than a specific model, so your settings
	do not change when you switch models:

	| Name | What it gives you |
	|---|---|
	| `waired/default` | The default model |
	| `waired/coding` | The coding model |
	| `waired/small` | Something fast and short |

3. **API key** — your tool will insist on one. Waired uses a key that lives on
	your own computer:

	<Tabs syncKey="os">
		<TabItem label="Linux">
			```sh
			cat ~/.config/waired/secrets/gateway-token
			```
		</TabItem>
		<TabItem label="macOS">
			```sh
			cat ~/Library/Application\ Support/waired/secrets/gateway-token
			```
		</TabItem>
		<TabItem label="Windows">
			```powershell
			Get-Content "$env:AppData\waired\secrets\gateway-token"
			```
		</TabItem>
	</Tabs>

	Paste that value in as the API key.

</Steps>

<Aside type="caution" title="That file is a password">
	Anything holding it can use your AI. It is readable only by you — keep it
	that way, and do not paste it into a bug report or a screenshot.
</Aside>

<Aside type="note" title="No key file yet?">
	It is created the first time a coding tool is connected. Run `waired link` to
	create it, or [`waired doctor`](/getting-started/doctor/), which reports the
	exact path it is using on your machine and creates it when you press **f**.
</Aside>

## Check it before configuring anything

```sh
curl http://127.0.0.1:9473/v1/models \
  -H "Authorization: Bearer $(cat ~/.config/waired/secrets/gateway-token)"
```

<Expected>
	A JSON list including `waired/default`. Those are exactly the names your
	chat app can use.
</Expected>

<Expected tone="bad">
```
curl: (7) Failed to connect to 127.0.0.1 port 9473
```
	Waired is not running, or routing is paused. Run
	[`waired doctor`](/getting-started/doctor/).
</Expected>

A full request looks like this:

```sh
curl http://127.0.0.1:9473/v1/chat/completions \
  -H "Authorization: Bearer $(cat ~/.config/waired/secrets/gateway-token)" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "waired/default",
    "messages": [{"role": "user", "content": "Say hi"}]
  }'
```

## Run the app on the same computer

The address `127.0.0.1` means "this computer only". It is deliberately not
reachable from the network — that is what keeps your AI from being exposed by
accident.

So a chat app in a browser must run on the **same computer** as Waired. If you
want to use your AI from a different machine, install Waired there too and
[sign in with the same account](/getting-started/add-a-device/) — the two
connect privately, and the chat app then talks to `127.0.0.1` on that machine.

<Aside type="caution" title="Setting these up by hand is still rough">
	Claude Code, OpenCode and OpenClaw are configured for you. Everything else
	works with the settings above, but you have to copy the key across yourself.
	The address and the model names are stable, so what you set up now will keep
	working.
</Aside>

## The full endpoint list

| API | Endpoint |
|---|---|
| OpenAI chat | `POST /v1/chat/completions` |
| OpenAI models | `GET /v1/models` |
| OpenAI responses | `POST /v1/responses` |
| Anthropic messages | `POST /anthropic/v1/messages` |

Other model names, and the specific model each Waired name resolves to, are in
the [model catalog](/reference/model-catalog/).
